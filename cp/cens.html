I actually had already done this exact problem for my Computer Science A class. While I had already used the fastest solution when I did the lab in that class, the most common solution that others in my class used was not up to speed. One might be tempted to do this problem by making the condition of a while loop while (string.indexOf(censor) != -1), however this method is incredibly slow as indexOf is a slow function. Instead, it is much faster to build a new string and continually check the end of it for the string that needs to be censored. One issue that I had when implementing this is that .substr() was too slow for this problem. Instead, I had to use .resize(). Here's the code for this problem:

string ans;
for (int i = 0; i < (int)s.size(); i++)
{
    ans += s[i];
    if (ans.size() >= t.size())
    {
        if (ans.substr(ans.size() - t.size()) == t)
        {
            ans.resize(ans.size() - t.size());
        }
    }
}
cout << ans << "\n";
