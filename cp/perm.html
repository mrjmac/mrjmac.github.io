This is the fifth problem in the CSES competitive programming problem set.

The problem goes like this: you are given a number. If it is possible, construct a beautiful permutation for the number. Otherwise print "NO SOLUTION".

In order to solve this problem, I first had to learn what a beautiful permutation is. Thankfully, CSES was kind enough to give an explanation: "A permutation of integers 1, 2, â€¦, n is called beautiful if there are no adjacent elements whose difference is 1."

Now that I knew what a beautiful permutation was, this problem seemed incredibly easy. However, this is probably the most frustrating problem I've had to do so far. The logic behind the code is incredibly simple, and my solving method never changed throughout the entire process. If you think about it, a beautiful permutation will always exist for every number besides 2 and 3. This is because even and odd numbers are never adjacent, and after 3 there are enough of these to always have a layer of separation. My code first checks if the number is 2 or 3, and if it is, it outputs "NO SOLUTION". After that, we just have to print out a beautiful permutation, which will always exist. I used two for loops to print out the even and the odd numbers up until n. It's important to print out the even numbers first, as if you print out the odd numbers first when n = 4, you'll get 1 3 2 4 which is not a beautiful permutation as 2 and 3 are adjacent.

However, submitting my code is when issues arose. No matter what changes I made, I could not get under the time limit on the last 9 cases. I looked online after 30 minutes of trying to find an issue with my solving method, but I could not find an alternative way to do this problem. Eventually I caved and rewrote my code in C++, and managed to get under the time limit with extreme ease, only using .1/1 seconds. Here's the code for this problem:

if (num == 3 || num == 2)
{
    cout << "NO SOLUTION" << endl;
}
else
{
    for (int i = 2; i <= num; i += 2)
    {
        cout << i << " ";
    }
    for (int i = 1; i <= num; i += 2)
    {
        cout << i << " ";
    }
}

I wanted to figure out how to do this with Java, so I stopped using Scanner and changed to using BufferedReader, PrintWriter, and StringTokenizer. Now I've gone back and reworked all my old problems using these to adjust to using them, but my solution worked fine with these instead of scanner. Here's the Java code for this problem:
