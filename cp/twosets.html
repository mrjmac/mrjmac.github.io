Two Sets

GitHub link

This is the eighth problem in the CSES competitive programming problem set.

The problem goes like this: given an integer n, divide the number into two sets of an equal sum. If it's not possible to do so, print "NO".

The trick to this problem is to break it down and re-examine what the question is asking. What matters here is that it's possible for every number up to n to be added up into two equal groups. By finding the sum of all natural numbers up until n using the formula (n * (n + 1)) / 2 find the remainder of this value when divided by 2, we can determine if it's possible to accomplish this task for given value n. This is because if the sum of all natural numbers isn't even, it can't be divided into two equal groups.

Now that we know it's possible to create two even groups, we have to figure out how to go about this. If you look at a list of numbers, you'll always noticed that the sum of the 1st number and the 4th number is equal to the sum of the 2nd number and the 3rd number. This means that if n % 4 == 0, we can simply loop up until n printing out every 1st and 4th number and repeat this with every 2nd and 3rd number. Now we have to figure out what to do with all values where n % 4 isn't 0. Values of 1 will never get past our summation filter, as a remainder of 1 when divided by 4 means that there is an additional odd number. An odd number cannot be divided evenly into two groups. The same therefore applies to values of 2, as an even plus an odd number is never even. However, when n % 4 is 3, it is possible to create a set. This is because the 1 + 2 equals 3. This handles the 3 remaining numbers and after that n % 4 will come out to 0 which means we can use the 1 + 4 = 2 + 3 method for our remaining numbers. Here's the code for this problem:
