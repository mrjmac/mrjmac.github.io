This is an old usaco bronze problem that I found through usaco.guide.

The problem is rather long like all usaco problems, so here's a link to it so you can read it.

This problem is pretty simple and simply parsing through the input will result in an O(n) solution. All we have to do is keep track of our current letter count and if we are over the limit we move the next word to a new line. In order to avoid adding spaces to the end of a line, we have to make sure that the next word fits before we add one. However, this can cause of to go out of bounds on the final word. To prevent this, I added a short circuit or statement so that the compiler will never attempt to access an out of bounds index. Here's the code for this problem:

String[] words = r.readLine().split(" ");
int currChar = 0;

for (int i = 0; i < n; i++)
{
    if (currChar + words[i].length() <= k)
    {
        currChar += words[i].length();
        if (i == n - 1 || currChar + words[i + 1].length() > k)
        {
            pw.print(words[i]);
        }
        else
        {
            pw.print(words[i] + " ");
        }
    }
    else
    {
        currChar = words[i].length();
        if (i == n - 1 || currChar + words[i + 1].length() > k)
        {
            pw.print("\n" + words[i]);
        }
        else
        {
            pw.print("\n" + words[i] + " ");
        }
    }
}
