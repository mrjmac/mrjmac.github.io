This is an old usaco bronze problem that I found through usaco.guide.

This problem is rather long like all usaco problems, so here's a link so you can read it.

This problem is a great reminder of why you should slow down and look for the easiest attempt and not the first one that you think of. USACO tricks you by listing all possibilities of the board in the output explanation. This led me to think that the easiest way to solve the problem was to use recursive backtracking to find every possible combination of words and then figure out how many letters I needed. However, this becomes an awful approach once you realize that only one of the two words can be shown at a time. The fact that this is true means that we can simply go through all the letters in both given words and add the maximum number of occurrences for each letter in the 2 given words to our answers. If we do this for all blocks it will yield the minimum number of blocks needed. Here's the code for this problem: 

for (int i = 0; i < n; i++)
{
    string word1, word2;
    cin >> word1 >> word2;
    vector<int> freq1(26), freq2(26);
    for (char c : word1)
    {
        freq1[c - 'a']++;
    }

    for (char c : word2)
    {
        freq2[c - 'a']++;
    }

    for (int i =  0; i < 26; i++)
    {
        ans[i] += max(freq1[i], freq2[i]);
    }
}
