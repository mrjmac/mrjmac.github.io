Two Tables

GitHub link

This is an introductory codeforces problem that I found through usaco.guide.

This problem is rather complicated and easier to understand with visuals like all 2D geometry problems, so here's a link so you can read it.

This problem is fairly simple and just comes down to understanding the situation. We can either put the second table on the top, bottom, left, or right side. It's pretty easy to figure out the distance that the first table needs to move for the second table to fit for all of these cases. All we have to do in in this problem is check if the second table will fit in at least one scenario. If it does, we just have to check which scenario requires the least movement. Here's the code for this problem

for (int i = 0; i < n; i++)
{
    int tw, th, x1, y1, x2, y2, w2, h2;
    cin >> tw >> th >> x1 >> y1 >> x2 >> y2 >> w2 >> h2;

    int w1 = x2 - x1;
    int h1 = y2 - y1;

    int leftP = w2;
    int rightP = tw - w2;
    int topP = th - h2;
    int bottomP = h2;

    int left = max(leftP - x1, 0);
    int right = max(x2 - rightP, 0);
    int top = max(y2 - topP, 0);
    int bottom = max(bottomP - y1, 0);

    if (w1 + w2 > tw)
    {
        left = INT32_MAX;
        right = INT32_MAX;
    }

    if (h1 + h2 > th)
    {
        top = INT32_MAX;
        bottom = INT32_MAX;
    }

    int ans = min(min(left, right), min(top, bottom));

    if (ans == INT32_MAX)
    {
        cout << -1 << "\n";
    }
    else
    {
        cout << ans << "\n";
    }
}
